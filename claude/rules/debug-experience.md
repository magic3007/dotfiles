# Debug经验保存规则

## 概述

本规则指导在调试过程中如何系统性地记录和保存调试经验，以便未来参考和知识积累。

## 规则内容

### 1. 保存时机
- 当解决一个非显而易见的bug时
- 当发现一个重要的调试技巧或模式时
- 当遇到需要多次尝试才能解决的问题时
- 当发现文档未覆盖的特殊情况时

### 2. 保存位置
调试经验应保存在当前项目的`debug_experience`目录下：
```
项目根目录/debug_experience/YYYY-MM-DD_问题描述.md
```

如果项目没有`debug_experience`目录，应自动创建。

### 3. 文件命名格式
```
YYYY-MM-DD_简短问题描述.md
```
示例：`2026-03-04_LATTE_compilation_issues.md`

### 4. 文档结构
每个调试经验文档应包含以下部分：

```markdown
# [问题标题]

**日期**: YYYY-MM-DD
**相关组件/模块**: [组件名称]
**状态**: [已解决/部分解决/未解决]

## 问题描述
[清晰描述遇到的问题]

## 症状与错误信息
[具体的错误消息、异常行为等]

## 根本原因分析
[分析问题的根本原因]

## 解决方案
[详细的解决步骤]

## 验证方法
[如何验证解决方案有效]

## 关键学习经验
[从这次调试中学到的经验教训]

## 预防措施
[如何预防类似问题再次发生]

## 相关资源
[相关的代码文件、配置、文档链接等]
```

### 5. 自动化建议
建议通过以下方式自动化此过程：
- 在Claude Code中设置PostToolUse hook，检测调试相关对话
- 当检测到调试活动结束时，自动提示用户保存经验
- 提供模板和自动填充部分信息

### 6. 质量要求
- 文档应具体、准确、可操作
- 包含足够的上下文信息
- 避免敏感信息泄露
- 定期回顾和更新过时的经验

## 实施指南

### 对于Claude Code用户
1. 将此规则文件放在`~/.claude/rules/debug-experience.md`
2. 在调试会话中，主动应用此规则
3. 在调试完成后，使用此模板记录经验

### 对于项目维护者
1. 在项目根目录创建`debug_experience`目录
2. 将重要的调试经验纳入项目文档
3. 定期团队回顾调试经验
